Versão 3.0 — Maio 2026 · 15 ferramentas + sistema de oficina · PROFNIT/UFRR

O que mudou na v3.0

🔄 Registro de Mudanças — v2.0 → v3.0

  • Removido: Crazy 8s da PI — a técnica de brainstorm rápido não se mostrou eficaz no contexto de PI, onde os participantes ainda não possuem repertório suficiente para gerar 8 estratégias diversas em 8 minutos. Na prática, os grupos produziam variações superficiais ou travavam.
  • Promovida: Busca de Anterioridade — elevada de "ferramenta de apoio" para "Recomendada" na Fase 3. A busca prática em bases reais (INPI, PatentScope) é a habilidade mais concreta que o participante pode levar da oficina.
  • Renomeada: Fase 4 de "Prototipar" para "Testar" — alinha com o que realmente acontece (simulação + debate + validação), sem criar expectativa de prototipagem física.
  • Reposicionado: PI Quest — movido de ferramenta opcional da Fase 5 para atividade assíncrona pós-oficina. O aluno acessa pelo Módulo do Aluno depois. Evita competição de tempo com o Escape Room.
  • Novo: Sistema de Oficina — Dashboard do Facilitador (dashboard.html) + Módulo do Aluno (aluno.html) com acompanhamento em tempo real via Firebase, ranking, controle de fases e envio de resultados.
  • Reorganizado: Storyboard agora precede o Canvas de Estratégia como rascunho narrativo (não mais como ferramenta independente paralela). A sequência na Fase 3 fica: Busca → Storyboard → Canvas.

Justificativa pedagógica

A versão 3.0 corrige três problemas identificados na aplicação prática da v2.0. O primeiro é a presença do Crazy 8s — uma técnica originada no Design Sprint do Google para geração rápida de soluções visuais de interface. No contexto de PI, os participantes estão no início do aprendizado e não possuem o domínio do espaço de soluções que a técnica exige. A busca de anterioridade prática é mais valiosa nesse ponto do percurso porque ensina uma habilidade concreta (consultar bases de dados) e alimenta diretamente a estratégia.

O segundo problema era a redundância entre Storyboard e Canvas de Estratégia — ambos cobriam planejamento de proteção, mas de formas diferentes. A solução foi sequenciá-los como um par complementar: o Storyboard como rascunho narrativo rápido, seguido pelo Canvas como formalização estruturada.

O terceiro era a ausência de documentação do sistema digital de oficina, que agora está totalmente integrado ao guia.

Sistema de Oficina — Visão Geral

Novidade da v3.0
O toolkit agora inclui um sistema completo de acompanhamento em tempo real. O facilitador projeta o Dashboard na tela da sala e acompanha o progresso de cada participante. Os alunos usam o Módulo do Aluno em seus celulares ou notebooks. Os dados trafegam via Firebase Realtime Database.

Arquitetura

O sistema é composto por 3 camadas que se comunicam pelo Firebase:

┌─────────────────────────────────────────────────────┐ │ GITHUB PAGES │ │ (hospedagem estática — todos os arquivos HTML/JS) │ ├──────────────┬──────────────┬───────────────────────┤ │ index.html │ aluno.html │ dashboard.html │ │ (vitrine │ (módulo do │ (painel do │ │ pública) │ aluno) │ facilitador) │ ├──────────────┴──────────────┴───────────────────────┤ │ │ │ 16 ferramentas .html │ │ Cada uma com tool-bridge.js integrado │ │ Modo livre (via index) OU modo oficina (via aluno) │ │ │ ├──────────────────────────────────────────────────────┤ │ FIREBASE REALTIME DB │ │ │ │ oficinas/[CODIGO]/ │ │ config/ → percurso, facilitador │ │ controle/ → fases_liberadas, ferramentas │ │ participantes/ → nome, status, scores, progresso │ │ [ID]/diagnostico/ → score, respostas │ │ [ID]/final/ → score, respostas │ │ [ID]/ferramentas/ → dados por ferramenta │ └──────────────────────────────────────────────────────┘

Fluxo durante a oficina

Facilitador cria a sessão Acessa dashboard.html, informa código da oficina (ex: "UFRR-2026-05"), seu nome e o percurso. Clica "Criar / Entrar na Oficina". O dashboard fica projetado na tela da sala.
Alunos entram na oficina Acessam aluno.html no celular/notebook. Informam nome, código da oficina e percurso. A interface mostra a lista de ferramentas com a primeira desbloqueada.
Aluno inicia uma ferramenta Clica "Iniciar" no Módulo do Aluno. É redirecionado para a ferramenta com parâmetros ?modo=oficina&tool=ID. Uma barra de contexto aparece no topo: "Oficina UFRR-2026-05 · Ferramenta: Quiz Diagnóstico".
Aluno completa a ferramenta Ao terminar, clica "Concluir e Enviar" (botão que aparece apenas em modo oficina). Os dados são enviados ao Firebase. Um overlay confirma "Dados enviados com sucesso!" e redireciona ao Módulo do Aluno.
Dashboard atualiza em tempo real O facilitador vê o ranking atualizar instantaneamente: scores do Quiz Diagnóstico, progresso por ferramenta, evolução. Sem necessidade de recarregar a página.
Facilitador pode liberar fases Na aba "Controle" do dashboard, o facilitador liga/desliga fases para todos os participantes. Se liberar a Fase 3, todos os alunos ganham acesso às ferramentas da Fase 3 mesmo sem ter completado a Fase 2.

Cada ferramenta funciona em dois modos, sem necessidade de arquivos separados:

Dashboard do Facilitador

Novo

Dashboard — dashboard.html

📊 Tempo real🏆 Ranking🎛️ Controle de fases📥 Exportação CSV

Como acessar

Abra no navegador: https://[seu-usuario].github.io/[repositorio]/dashboard.html. Não há restrição de acesso — qualquer pessoa com a URL pode abrir. Na oficina, projete na tela principal da sala.

4 abas do dashboard

Aba Ranking — Exibe ranking em tempo real com três visualizações: Quiz Diagnóstico (score D-X), Quiz Final (score F-X) e Evolução (diferença F-X menos D-X). Cada participante aparece com posição, nome, score e medalha. Ideal para projetar na tela durante toda a oficina — cria engajamento e competição saudável.

Aba Participantes — Tabela completa com status de conexão (ativo, desconectado, saiu), nome, percurso, scores D-X e F-X, evolução, progresso percentual e indicadores visuais (dots coloridos) para cada uma das 15 ferramentas. Permite ao facilitador identificar quem está travado.

Aba Análise — Quatro gráficos de barras: conclusão por ferramenta (quantos participantes completaram cada uma), taxa de erro por questão do Quiz Diagnóstico (identifica conceitos que a turma não domina), distribuição de TRL selecionado no Canvas de Diagnóstico, e média de estrelas por fase no Canvas de Avaliação.

Aba Controle — Toggles para liberar ou bloquear cada uma das 5 fases. Quando o facilitador ativa uma fase, todos os participantes recebem acesso às ferramentas correspondentes, independente do progresso individual. Ao liberar a Fase 3, as Fases 1 e 2 são automaticamente liberadas também.

Funcionalidades extras

O botão "CSV" exporta todos os dados dos participantes em formato planilha. O botão "Imprimir" gera versão otimizada para papel (A4 paisagem). Os cards de estatísticas no topo mostram: total de participantes, ativos agora, média D-X, média F-X, evolução média e percentual de conclusão geral.

Uso pedagógico do ranking
Projete o ranking na tela durante o Quiz Diagnóstico para criar expectativa. Depois, na hora do Quiz Final, projete novamente — os participantes ficam ansiosos para ver sua evolução. A coluna "Evolução" (diferença entre F-X e D-X) é mais significativa que o score absoluto. Celebre quem mais evoluiu, não apenas quem tirou a nota mais alta.

Módulo do Aluno

Novo

Módulo do Aluno — aluno.html

📱 Responsivo🔓 Desbloqueio progressivo📊 Progresso visual💾 Funciona offline

Como acessar

Alunos acessam: https://[seu-usuario].github.io/[repositorio]/aluno.html. Na tela de login, informam nome completo, código da oficina (fornecido pelo facilitador) e percurso (o facilitador diz qual selecionar).

Interface do aluno

Após login, o aluno vê uma lista vertical de todas as ferramentas do percurso selecionado, organizadas por fase. Cada ferramenta mostra nome, ícone, duração estimada, classificação (Essencial/Recomendada/Opcional) e um dos seguintes estados:

Próxima (destaque azul pulsante + botão "Iniciar") — é a ferramenta que o aluno deve fazer agora. Apenas uma por vez está neste estado.

Desbloqueada (borda azul + botão "Abrir") — disponível mas não é a próxima na sequência. Aparece quando o facilitador libera fases manualmente.

Concluída (fundo verde + check) — aluno já completou e enviou os dados.

Bloqueada (cinza + cadeado) — ainda não disponível. Será desbloqueada quando o aluno concluir as anteriores ou quando o facilitador liberar a fase correspondente.

No topo, um círculo de progresso mostra a porcentagem de conclusão. A barra linear abaixo replica o progresso com gradiente colorido.

Desbloqueio dual

As ferramentas são desbloqueadas por dois mecanismos independentes (o que ocorrer primeiro):

Por progressão sequencial: Ao completar uma ferramenta, a próxima na lista é desbloqueada automaticamente.

Por liberação do facilitador: Quando o facilitador ativa uma fase no Dashboard (aba Controle), todas as ferramentas daquela fase ficam disponíveis para todos os participantes.

Modo offline

Se o aluno perder conexão com a internet, o Módulo do Aluno continua funcionando com dados salvos em localStorage. Ao reconectar, os dados são sincronizados automaticamente com o Firebase. O indicador de conexão na tela de login mostra "Conectado ao servidor" ou "Modo offline — dados salvos localmente".

Instrução para os alunos — Abertura da oficina

"Abram o navegador no celular e acessem este endereço [projetar URL ou QR code]. Informem o nome de vocês e o código da oficina que está na tela: [CÓDIGO]. No campo percurso, selecionem [2h/3h/4h]. Quando aparecer a lista de ferramentas, a primeira já vai estar disponível. Sigam na ordem — cada vez que vocês terminam uma ferramenta e clicam 'Concluir e Enviar', a próxima desbloqueia automaticamente."

Configuração do Firebase

O sistema usa o Firebase Realtime Database para comunicação em tempo real entre alunos e facilitador. A configuração já está embutida no arquivo js/firebase-config.js. Se precisar criar seu próprio projeto Firebase:

Criar projeto no Firebase Console Acesse console.firebase.google.com. Crie um novo projeto. Ative o Realtime Database na região us-central1.
Configurar regras de acesso Em Realtime Database → Rules, cole: 
{
  "rules": {
    "oficinas": {
      "$oficina": {
        ".read": true,
        ".write": true
      }
    }
  }
}

Para produção, restrinja com autenticação. As regras acima são para desenvolvimento e oficinas controladas.

Obter credenciais Em Project Settings → General → Your apps → Web app, copie o objeto firebaseConfig e substitua no arquivo js/firebase-config.js.
Testar Abra dashboard.html e aluno.html em abas diferentes. Crie uma sessão no dashboard, entre com o aluno. Os dados devem aparecer em tempo real no dashboard.
Sobre segurança
As credenciais do Firebase no arquivo firebase-config.js são públicas por design — o Firebase é projetado para expor essas chaves no client-side. A segurança é controlada pelas Security Rules do Realtime Database, não pela ocultação das credenciais. Para oficinas de curta duração com público controlado, as regras abertas são aceitáveis. Para uso contínuo, implemente regras mais restritivas com autenticação.
🔍 Fase 1 — Descobrir

Objetivo da Fase

Mapear conhecimento prévio, despertar curiosidade e criar linha de base (score D-X). O facilitador atua como mediador socrático: faz perguntas, acolhe respostas sem julgamento. Evite corrigir erros neste momento.

Duração: 25 min (2h) a 45 min (4h). Ferramentas: Quiz Diagnóstico (E), Cartas Personas (E), Mapa de Empatia (R).

Abertura da Fase 1

"Vamos começar com um diagnóstico rápido. Quero descobrir o que vocês já sabem sobre Propriedade Intelectual. No Módulo do Aluno de vocês, a primeira ferramenta já está desbloqueada — é o Quiz Diagnóstico. Anotem o código D-X que vão receber no final."

Integração com o sistema
Quando o aluno completa o Quiz Diagnóstico pelo Módulo do Aluno, o score D-X aparece automaticamente no ranking do Dashboard. Projete o ranking na tela após todos terminarem — gera engajamento imediato.
Essencial

1.1 — Quiz Diagnóstico

⏱ 10–15 min👤 Individual💻 quiz-diagnostico.html

10 questões de múltipla escolha. Gera código D-X. No modo oficina, o score e as respostas individuais são enviados ao Firebase, permitindo que o Dashboard exiba o ranking em tempo real e a análise de erros por questão.

Dados enviados ao Dashboard: score, total, código D-X, tempo gasto, e para cada questão: alternativa selecionada, alternativa correta, se acertou, categoria do conceito.

Essencial

1.2 — Cartas Personas

⏱ 10–15 min👥 Grupos 3–5💻 cartas-personas.html

10 cartas com perfis do ecossistema de PI amazônico. No modo oficina, a persona sorteada é registrada no Firebase para que o facilitador saiba qual grupo trabalha com qual perfil.

Recomendada

1.3 — Mapa de Empatia do Inventor

⏱ 15 min👥 Grupos 3–5💻 mapa-empatia.html

Canvas com 6 quadrantes. No modo oficina, os post-its de cada quadrante são enviados ao Firebase. O Dashboard pode exibir as dores mais mencionadas na aba Análise.

🧭 Fase 2 — Diagnosticar

Objetivo da Fase

Classificar criações nos tipos corretos de PI e compreender o contexto. O facilitador muda para instrutor ativo — é o momento de transmitir conhecimento estruturado.

Duração: 25 min (2h) a 40 min (4h). Ferramentas: Canvas de Diagnóstico (E), Árvore de Decisão (E), Linha do Tempo (O).

Essencial

2.1 — Canvas de Diagnóstico de PI

⏱ 15 min👥 Grupos💻 canvas-diagnostico.html

8 campos incluindo slider TRL (1–9) e seleção múltipla de tipos de PI. No modo oficina, envia: todos os campos de texto, valor TRL, tipos selecionados. O Dashboard usa o TRL para o gráfico de distribuição na aba Análise.

Essencial

2.2 — Árvore de Decisão

⏱ 10–15 min👥 Individual ou Grupos💻 arvore.html

Wizard sim/não que identifica o tipo de proteção entre 10 modalidades. No modo oficina, o resultado final é enviado ao Firebase.

Opcional

2.3 — Linha do Tempo da PI no Brasil

⏱ 10 min👥 Plenária💻 linha-do-tempo.html

16 marcos históricos de 1809 a 2023. Use como tour guiado rápido — 5 a 6 marcos essenciais são suficientes.

📐 Fase 3 — Estrategar

Objetivo da Fase

Transformar diagnóstico em plano de ação. A sequência é intencional: Busca de Anterioridade (descobrir o que já existe) → Storyboard (narrar o caminho de proteção) → Canvas de Estratégia (formalizar o plano com custos e prazos reais).

Duração: 15 min (2h, só Canvas) a 40 min (4h, sequência completa).

Por que o Crazy 8s foi removido
A técnica Crazy 8s exige que o participante já domine o espaço de soluções para gerar alternativas rapidamente. Em PI, neste ponto da oficina os participantes ainda estão construindo repertório — forçar 8 estratégias em 8 minutos produzia repetições superficiais. A Busca de Anterioridade prática é mais valiosa porque ensina uma habilidade concreta e alimenta diretamente o Canvas de Estratégia.
Recomendada

3.1 — Busca de Anterioridade (atividade prática)

⏱ 15 min👥 Grupos💻 busca.html

Anteriormente classificada como "ferramenta de apoio", agora é uma atividade estruturada. Os grupos usam o Painel de Busca Guiada para fazer uma busca real nas bases do INPI e PatentScope para a criação da persona deles.

Contextualizar (3 min)
Você pode dizer

"Antes de proteger, precisamos saber o que já existe. Imaginem depositar uma patente e descobrir 2 anos depois que alguém já registrou a mesma coisa — vocês teriam desperdiçado tempo e dinheiro. A busca de anterioridade é como pesquisar no Google antes de comprar um domínio. Vamos fazer isso agora, de verdade."

Busca guiada (10 min) Cada grupo abre busca.html e segue o mini-tutorial para o tipo de PI da persona. Para patentes: BuscaWeb do INPI ou PatentScope. Para marcas: base de marcas do INPI. Devem encontrar pelo menos 2 resultados relacionados e anotar no Canvas de Estratégia (bloco 9 — Riscos).
Compartilhamento (2 min) Cada grupo relata: "Encontramos X resultados similares. O diferencial da nossa criação é Y."
Recomendada

3.2 — Storyboard de Proteção

⏱ 10–15 min👥 Grupos💻 storyboard.html

Narrativa visual em 6 cenas. Funciona como rascunho narrativo que antecede o Canvas de Estratégia. Os grupos contam a história da proteção antes de formalizá-la.

Você pode dizer

"Antes de preencher o plano formal, contem a história da proteção da criação de vocês em 6 cenas — como um mini-filme. A cena 2 é a mais importante: é onde o 'quase-erro' acontece. Isso vai ajudar vocês a enxergarem o caminho completo antes de formalizar."

Essencial

3.3 — Canvas de Estratégia de PI

⏱ 15 min👥 Grupos💻 canvas-estrategia.html

O produto principal da Fase 3. 10 blocos com auto-preenchimento de custos, prazos e legislação ao selecionar o tipo de PI. Os blocos 3–7 são preenchidos automaticamente. Os grupos focam nos blocos 8 (stakeholders), 9 (riscos — incluindo resultados da busca de anterioridade) e 10 (plano de ação com 5 passos concretos).

Integração com o sistema
No modo oficina, o Canvas de Estratégia envia ao Firebase: tipo de PI selecionado, tipo complementar, texto de stakeholders, riscos identificados e todas as ações do plano (ação + responsável + prazo). O facilitador pode ver no Dashboard quais tipos de PI os grupos estão escolhendo.
🔬 Fase 4 — Testar

Objetivo da Fase

Confrontar a estratégia com a realidade: simular depósito, debater dilemas éticos, validar conhecimento. O nome "Testar" (e não mais "Prototipar") reflete melhor o que acontece: não há prototipagem física, mas sim teste e validação das estratégias formuladas na Fase 3.

Duração: 35 min (2h) a 55 min (4h). Ferramentas: Simulador de Depósito (E), Baralho de Dilemas (E), Escape Room (O).

Essencial

4.1 — Simulador de Depósito

⏱ 20–30 min👥 Grupos💻 simulador-deposito.html

Simula o processo completo de depósito para 9 tipos de proteção. No modo oficina, envia o tipo selecionado e se aplicou desconto de 50%.

Essencial

4.2 — Baralho de Dilemas

⏱ 10–15 min👥 Plenária💻 baralho-dilemas.html

16 cartas com dilemas éticos e estratégicos. No modo oficina, envia: quantidade discutida e anotações por dilema.

Opcional

4.3 — Escape Room da PI

⏱ 15 min (curta)👥 Grupos💻 escape-room.html

8 puzzles temáticos com timer. Para o percurso de 4h, use versão curta (puzzles 1–3). No modo oficina, envia: score, dicas usadas, tempo.

🔄 Revisão Estratégica (10 min) — apenas percurso 4h

Após o Simulador e os Dilemas, os participantes reabrem o Canvas de Estratégia e atualizam os blocos 9 (Riscos) e 10 (Plano de Ação) com o que aprenderam. Esse loop iterativo é o que diferencia uma sequência linear de uma experiência de Design Thinking genuína.

Você pode dizer

"Voltem ao Canvas de Estratégia. Depois de simular o depósito e debater os dilemas, o que mudaria no plano de vocês? Atualizem pelo menos um risco novo e uma ação nova."

📊 Fase 5 — Evoluir

Objetivo da Fase

Avaliar aprendizagem, comparar D-X com F-X, consolidar conhecimentos e orientar próximos passos.

Duração: 20 min (2h) a 25 min (4h). Ferramentas: Quiz Final (E), Canvas de Avaliação (R).

Essencial

5.1 — Quiz Final

⏱ 10–15 min👤 Individual💻 quiz-final.html

20 questões avançadas. Gera código F-X. No modo oficina, o score é enviado ao Firebase e o ranking do Dashboard atualiza instantaneamente — este é o momento mais impactante da oficina. Projete o ranking na tela e compare as evoluções.

Momento de projeção do ranking
Quando os últimos alunos terminarem o Quiz Final, mude o Dashboard para a visualização "Evolução". A turma toda verá quem mais evoluiu do D-X para o F-X. Celebre quem teve maior evolução, não apenas quem tirou a nota mais alta. Um aluno que foi de D-2 para F-8 (+6) evoluiu mais que um que foi de D-7 para F-9 (+2).
Recomendada

5.2 — Canvas de Avaliação

⏱ 10 min👤 Individual💻 canvas-avaliacao.html

Avaliação de cada fase com estrelas (1–5), comentários por fase, e campos de pontos fortes, melhorias, aprendizado principal e próximos passos. No modo oficina, as estrelas aparecem no gráfico "Avaliação por Fase" na aba Análise do Dashboard.

Apêndice D — Adaptações e Variantes

Oficina 100% online (síncrona)

Compartilhe o link do aluno.html pelo chat. Use breakout rooms para trabalhos em grupo. O Dashboard funciona identicamente online. Aumente a duração em 15–20%.

Sem internet / modo offline

Copie todos os arquivos para um servidor local ou pen drive. As ferramentas funcionam 100% offline (apenas o Firebase não estará disponível). Nesse caso, use o modo livre via index.html e colete dados manualmente.

Versão assíncrona / autoestudo

O aluno acessa aluno.html, cria uma "oficina individual" com código pessoal, e segue o Percurso Compacto no próprio ritmo. O PI Quest é especialmente valioso nesse contexto.

Apêndice E — Dicas de Facilitação

Gestão de tempo

Use o Dashboard projetado como cronômetro visual — quando os alunos veem o progresso dos colegas, se auto-regulam. Regra: se precisa encurtar, reduza apresentações de grupo, não o trabalho individual.

Problemas técnicos

Plano B: cópias impressas dos Canvas em A3, versão oral do Quiz. Se o Firebase cair, as ferramentas continuam funcionando — apenas o ranking em tempo real fica indisponível.

Turma muito grande (+20 pessoas)

O Dashboard suporta qualquer número de participantes. Com 20+, tenha um cofacilitador para cada 10 adicionais. Use o Módulo do Aluno para que todos avancem no próprio ritmo, e sincronize com a turma apenas nos momentos de apresentação.
